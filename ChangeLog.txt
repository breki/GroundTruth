LATEST
------
- BUGFIX: better support for 0.6 OSM API
	- GT can now load OSM XML files which contain multiple versions of the same OSM object
	- all generated OSM objects now have versions
	- the OSM files should now be loadable (at least) in JOSM
- GT now generates zipped OSM files by default

1.7.735.16
----------
- BUGFIX: GT now uses its own temporary file naming to avoid problems with generating too many
	temp files in the user's home Temp directory

1.7.733.15
----------
- BUGFIX: lat/lon were switched for generated OSM files.... ooops
- BUGFIX: the app. will now not fall apart if the user specifies an area which is not covered by (SRTM) DEM data
- the application now uses "neutral" (U.S.) culture for displaying console messages (no more decimal commas)
- Flee library is no longer used

1.7.702.14
----------
- a workaround for missing contour tiles problem

1.6.318.2
---------
- coastline rendering improved
- map background color support
- added "extimeout" option
- using newer versions of Brejc* libraries

1.5.229.0
---------
- added SrtmServerUrl application setting to be able to switch to NASA's new URLs easily
- added DemSources application setting for the future support of various DEM sources
- temporary files generated by "contours" command are now deleted

1.5.205.6
---------
- added "-nocut" option

1.5.203.2
---------

1.5.128.2

1.2.20.0
--------

- BUGFIX: GT now requires that each OSM file has its own map name. 
- BUGFIX: the type IDs are now always written as hexadecimal values
- BUGFIX: problems with OSM objects which are used by more than one relation 
- BUGFIX: "DummyPath.img" is now always looked for in the GroundTruth's directory, not the current one
